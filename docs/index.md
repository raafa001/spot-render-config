# Spot Render Config – TechDocs

## Papel
- Centralizar blueprints, checklists e links dos repositórios.  
- Definir owners no Backstage (group + system).  
- Manter orientações sobre dados confidenciais (render lists).

## Conteúdo
- `blueprint-spot-render.md`
- `issues-plan.md`
- `backstage/catalog-info.yaml`

## Como usar
1. Atualize o blueprint sempre que a arquitetura mudar e gere nova versão no Backstage.  
2. Use `issues-plan.md` para abrir issues sincronizadas em cada repositório.  
3. Para render lists, documente aqui a localização e regras de acesso (não comitar arquivos).

## TechDocs
- `mkdocs.yml` exporta este documento para o Backstage.
